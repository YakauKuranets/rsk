# Архитектура системы (Tauri 2 + Rust + React)

> ⚠️ **Важно:** этот документ описывает целевую и текущую архитектуру на уровне проекта. Он **не является подтверждением 100% готовности всех модулей**. Фактическая готовность определяется кодом и статусом задач.

## Назначение

Приложение предназначено для автоматизированного сбора технических метрик, инвентаризации оборудования, контроля соответствия политикам безопасности и централизованного резервного копирования конфигураций/архивов с сетевых устройств в корпоративной инфраструктуре.

## 1. Общая схема

- **Frontend (React + Leaflet + Video.js)** — интерфейс оператора: карта, списки устройств, просмотр потоков, статус задач.
- **Backend (Rust + Tauri 2)** — системное ядро с доступом к локальным ресурсам, сетевым протоколам, процессам и зашифрованному хранилищу.
- **Локальные сервисы**:
  - обработка и проксирование видео (FFmpeg → HLS);
  - локальный HTTP-сервер потоков;
  - модули сканирования, сбора данных и синхронизации.

```
[React UI] <---- invoke/events ----> [Tauri Core (Rust)]
                                           |
                      +--------------------+--------------------+
                      |                    |                    |
                [Stream Engine]     [Discovery/Scanner]   [Storage/Vault]
                 FFmpeg + HLS        FTP/HTTP/RTSP APIs    Sled + AES-GCM
```

## 2. Ключевые компоненты

### 2.1 Системное ядро (Rust / Tauri)

- **Tauri v2**: desktop-runtime через системный WebView.
- **Точка входа**: `src-tauri/src/main.rs`.
- **Обмен с UI**: `invoke`-команды и события.

### 2.2 Хранилище и защита данных

- Локальная БД на **Sled**.
- Шифрование конфигураций/данных на **AES-256-GCM**.
- Привязка ключевого материала к идентификатору хоста для снижения риска офлайн-компрометации копии БД.

### 2.3 Видео-ядро

- **FFmpeg** для нормализации потоков RTSP/MJPEG.
- Локальная раздача в **HLS** для фронтенда.
- Воспроизведение через **Video.js**.

### 2.4 Сетевые и интеграционные модули

- HTTP-клиенты на `reqwest`.
- FTP/архивные операции через `suppaftp` (или кастомный TCP-клиент для специфичных серверов).
- Модули discovery/инвентаризации: определение сервисов по портам, базовый fingerprint устройств.

### 2.5 UI-ядро

- `src/App.jsx` — основной пульт оператора.
- Карта на **Leaflet** (в т.ч. кластеризация/фильтры для больших объёмов устройств).

## 3. Что реализовано сейчас (по факту кода)

- Команды Tauri для поиска и запуска потоков, FTP-браузинга и загрузки файлов.
- Хранилище на Sled и сериализация записей устройств.
- Интеграция внешнего поиска и базовый модуль сканирования.
- UI-панели для запуска потоков, поиска и работы с архивами.

## 4. Что в планах (не полностью реализовано)

1. **FTP-совместимость для нестандартных серверов**
   - обработка «медленных»/нестандартных баннеров;
   - настраиваемые таймауты/ретраи;
   - fallback-режим на уровне TCP.

2. **Углублённое извлечение архивов**
   - интеграция ISAPI/ONVIF для NVR/камер;
   - единый менеджер загрузок (прогресс, скорость, ошибки, возобновление).

3. **UX для масштаба**
   - фильтрация и группировка целей на карте;
   - встроенная панель live-логов backend.

## 5. Структура репозитория (основные точки)

- `src-tauri/src/main.rs` — orchestration backend и Tauri-команды.
- `src-tauri/src/videodvor_scanner.rs` — модуль сканирования/сбора данных.
- `src/App.jsx` — основная UI-композиция.

## 6. Нефункциональные требования

- TLS для сетевых интеграций, где применимо.
- Маскирование чувствительных данных в логах.
- Ротация ключей и ограничение времени жизни токенов доступа.
- Явная декларация и аудит разрешений приложения.
